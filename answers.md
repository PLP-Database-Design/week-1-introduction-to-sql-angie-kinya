## **Assignment Questions**

1. State and Explain the components of a DBMS(Database Management System). 

- Hardware: The physical devices like computers, storage devices, and network infrastructure that support the DBMS.
- Software: The DBMS software itself, which provides the tools and interfaces to manage the database.
- Data: The actual information stored in the database, organized in tables, records, and fields.
- Procedures: The set of rules and instructions for using the DBMS, including backup, recovery, and security protocols.
- Database Access Language: A language like SQL (Structured Query Language) used to interact with the database, perform queries, and manipulate data.
- Users: Individuals or applications that interact with the database, including database administrators, developers, and end-users.

2. What is a relational database? Give 4 examples.

- Relational Database: A type of database that stores data in tables (relations) consisting of rows and columns. It uses SQL for querying and managing data, and relationships between tables are established using keys.
- Examples: MySQL, PostgreSQL, Oracle Database, Microsoft SQL Server

3. State and Explain three classifications of SQL?

- Data Definition Language (DDL): Used to define and modify the structure of database objects like tables, indexes, and schemas. Examples: CREATE, ALTER, DROP.
- Data Manipulation Language (DML): Used to manage data within the database. Examples: INSERT, UPDATE, DELETE.
- Data Control Language (DCL): Used to control access to data in the database. Examples: GRANT, REVOKE.

4. What is the difference between a Primary Key and a Foreign Key?

- Primary Key: A unique identifier for each record in a table.It ensures no duplicate or null values. Only one primary key per table.
- Foreign Key: A field in one table that refers to the primary key in another table. Establishes a relationship between two tables. Can have duplicate or null values.

5. What is an Entity-Relationship Diagram?

- Entity-Relationship Diagram (ERD): A visual representation of the entities (tables) in a database and the relationships between them. It uses symbols like rectangles (entities), diamonds (relationships), and lines (connections) to depict the structure and design of the database.

6. What are the advantages of relational databases?

- Data Integrity: Ensures accuracy and consistency of data through constraints like primary keys and foreign keys.
- Flexibility: Easy to modify and extend the database structure without affecting existing data.
- Scalability: Can handle large amounts of data and support multiple users simultaneously.
- Ease of Use: SQL provides a simple and powerful way to query and manipulate data.
- Data Security: Offers robust security features to control access and protect data.

7. State four types of data type used to store data in tables?

- Integer: Used to store whole numbers (e.g., INT, BIGINT).
- String: Used to store text data (e.g., VARCHAR, CHAR).
- Date/Time: Used to store date and time values (e.g., DATE, TIMESTAMP).
- Boolean: Used to store true/false values (e.g., BOOLEAN).
   
8. What is the purpose of a database management system (DBMS)? 

- Data Storage and Retrieval: Efficiently stores and retrieves data for users and applications.
- Data Security: Protects data from unauthorized access and ensures privacy.
- Data Integrity: Maintains accuracy and consistency of data through constraints and rules.
- Data Sharing: Allows multiple users and applications to access and use the same data simultaneously.
- Backup and Recovery: Provides mechanisms to back up data and recover it in case of failure.
- Data Abstraction: Hides the complexity of data storage and provides a simplified interface for users.
